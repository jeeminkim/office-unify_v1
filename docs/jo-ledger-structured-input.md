# 조일현 구조화 원장 입력 (요약)

## 반영 엔진

- 허용: **INSERT**(적용 시 upsert), **DELETE**만.
- **UPDATE/SELECT는 불가**. “수정” UX도 동일 키(`market` + `symbol`, `user_key`는 서버 주입)로 **INSERT 한 줄**을 다시 보내 덮어쓴다.

## 부분 수정(보유)

- UI상 메모만·목표가만 등으로 줄여 보여도, validator는 INSERT에 필요한 필드를 요구한다.
- 그래서 **원장 스냅샷 GET**으로 기존 행을 불러와 수량·평단 등을 병합한 뒤 전송하는 흐름을 쓴다.
- 원장에 해당 시장/심볼 행이 없으면 부분 수정을 진행하지 말고 **전체 입력(신규)**으로 처리한다.

## 위원회와 사용자 메모

- 원장은 **사실**과 **사용자 메모(참고)**로 나뉘어 프롬프트에 들어간다.
- 위원회는 메모를 **결론처럼 재진술하지 않는다**. 구조·비중·전이·실행 우선순위를 앞에 둔다.
